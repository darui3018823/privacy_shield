# Universal Privacy Shield への貢献について

Universal Privacy Shield への貢献をご検討いただきありがとうございます！あなたの貢献は、このプロジェクトをより良いものにする助けとなります。

このドキュメントでは、バグ報告、機能提案、コード変更の送信など、Universal Privacy Shield への貢献に関するガイドラインを概説します。

## 行動規範 (Code of Conduct)

このプロジェクトは [Contributor Code of Conduct](Code_of_Conduct.md) の下で公開されています。このプロジェクトに参加することにより、その規約に従うことに同意したものとみなされます。

## 貢献する方法

### バグの報告

バグを見つけた場合は、GitHub で Issue を作成してください。新しい Issue を作成する前に、同様の Issue が既に存在しないか確認してください。バグを報告する際は、以下を含めてください：

*   バグの明確かつ簡潔な説明
*   現象を再現するための手順
*   期待される動作
*   該当する場合は、スクリーンショットまたはエラーメッセージ
*   使用しているOSとブラウザのバージョン

### 機能の提案

新しい機能の提案を歓迎します！GitHub の Issue として提出してください。機能強化を提案する際は、以下を含めてください：

*   機能の明確かつ簡潔な説明
*   なぜこの機能があなたや他の人にとって有用なのか
*   機能がどのように動作するかの例

### プルリクエスト (Pull Requests)

1.  リポジトリをフォークします。
2.  機能追加やバグ修正のための新しいブランチを作成します (`git checkout -b feature/amazing-feature`)。
3.  変更を加えます。
4.  変更をコミットします (`git commit -m 'Add some amazing feature'`)。
5.  ブランチにプッシュします (`git push origin feature/amazing-feature`)。
6.  プルリクエストを作成します。

コードが既存のコーディングスタイルに従っており、適切なコメントが含まれていることを確認してください。

## 開発のセットアップ

1.  リポジトリをクローンします。
2.  ブラウザに拡張機能を読み込みます（デベロッパーモード）。
3.  変更を加え、拡張機能をリロードしてテストします。

ご協力ありがとうございます！
